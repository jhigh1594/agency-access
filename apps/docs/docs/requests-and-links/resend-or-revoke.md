# Resend or Revoke a Request

Use resend when the request is still valid and the client simply needs another copy. Revoke when the underlying request should no longer be used.

## Resend when

- The client lost the original email.
- The request details are still correct.
- You are working with the same contact and same platform scope.

## Revoke when

- The wrong client or wrong permissions were requested.
- Your agency changed the requested platform list.
- The original link should no longer be usable for security or operational reasons.

## Safe workflow

1. Review the current request details.
2. Decide whether the request is still the source of truth.
3. Resend if the request is valid.
4. Revoke and create a replacement if anything material changed.

## Team note

Avoid sending multiple active links for the same job unless your team has a documented reason. Parallel links create confusion for both your client and your internal operators.
