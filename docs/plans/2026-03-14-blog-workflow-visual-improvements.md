# Blog Workflow Visual Content Improvements Plan

**Date:** 2026-03-14
**Status:** Planning

## Problem Statement

The two new blog posts generated by the automated workflow have excessive dashed line dividers (`---`) and lack the visual richness (tables, code blocks, structured sections) that make existing posts like the TikTok guide more engaging and scannable.

## Analysis: TikTok Blog Post Visual Elements

The TikTok blog post (`tiktok-ads-access-guide`) demonstrates effective visual content through:

### 1. Tables for Structured Information
- **Entity Limits Table:** Business Centers, Ad accounts, Campaigns limits
- **Permission Roles Tables:** Ad Account roles, Financial roles, Business Center roles
- **Platform Comparison Table:** TikTok vs Meta vs Google vs LinkedIn

### 2. Code Blocks for Visual Hierarchy
- **Account Hierarchy Tree Diagram:** ASCII art showing Business Center structure
- **Step-by-Step Numbered Lists:** Clear action items with sub-items

### 3. Structured Section Patterns
- **"Issue/Solution" Format:** For troubleshooting sections
- **"Pro Tip" Callouts:** Highlighted practical advice
- **"Why This Matters" Explanations:** Context for unusual requirements

### 4. Minimal Section Dividers
- **No `---` dividers** between major sections
- **Clear heading hierarchy** (H2 ##, H3 ###) provides structure
- **White space** as visual separator instead

### 5. Comparison Pattern
```
### Issue X: "Problem Title"

**Cause:** Root cause explanation.

**Solution:**
1. Step one
2. Step two
```

## Proposed Workflow Changes

### Change 1: Remove Dashed Line Dividers

**Location:** `.claude/prompts/blog-creation-workflow.md`

**Current Issue:** Workflow doesn't explicitly mention avoiding `---` dividers

**Solution:** Add to `<hard_constraints>` section:
```markdown
- NEVER use dashed line dividers (---) between sections
- Use clear heading hierarchy (H2 ##, H3 ###) instead
- Let white space create visual separation
```

### Change 2: Add Table Requirements

**Location:** New section in workflow prompt

**Add to category templates:** Each category template should include "Visual Elements to Include" section specifying:
- **Tutorials:** Platform comparison tables, permission role tables, step-by-step checklists
- **Onboarding:** Checklist tables (checkbox items), timeline tables
- **Comparisons:** Feature comparison tables, pricing tables
- **Security:** Risk assessment tables, permission level tables

**Example addition to workflow:**
```markdown
<visual_content_requirements>
**EVERY blog post MUST include at least 2-3 of the following visual elements:**

1. **Tables** - For structured data:
   - Platform/feature comparisons
   - Permission levels or role definitions
   - Checklists with checkbox formatting
   - Issue/Solution pairs
   - Step-by-step workflows with sub-items

2. **Code/Diagram Blocks** - For visual hierarchy:
   - ASCII art diagrams for account hierarchies
   - Code snippets for technical examples
   - Step-by-step numbered lists with sub-bullets

3. **Structured Sections** - For scannability:
   - Issue/Solution format for troubleshooting
   - Pro Tip callouts for practical advice
   - Why This Matters boxes for context
   - Key Takeaway summaries

**Table Formatting Examples:**

| Role | Permissions | Best For |
|------|-------------|----------|
| **Admin** | Full permissions | Campaign managers |

| Platform | Limit | Notes |
|----------|-------|-------|
| TikTok | 3 Business Centers | Plan carefully |

**Diagrams/Code Blocks:**
```
Business Center (max 3 per user)
    |
    +-- Ad Account 1
    +-- Ad Account 2
```
</visual_content_requirements>
```

### Change 3: Update Category Templates

**Location:** Update each category template with specific visual content requirements

**Example for Tutorials category:**
```markdown
### Tutorials (Platform Access Guides)
**Visual Elements to Include:**
- Platform comparison table (if multiple platforms mentioned)
- Permission level/role table
- Step-by-step workflow with numbered sub-items
- Common problems table (Problem | Solution | Prevention)

**Structure:**
1. Hook with data point
2. Why this platform matters
3. Platform-specific structure overview
4. **Table:** Permission levels or access requirements
5. Step-by-step: Manual process (numbered with sub-items)
6. Step-by-step: Using AuthHub (numbered with sub-items)
7. **Table:** Common issues & solutions
8. Pro tips
9. Security best practices
10. AuthHub CTA
```

## Implementation Plan

### Phase 1: Update Workflow Prompt (Safe Changes)
1. Add visual content requirements section
2. Update category templates with table examples
3. Add "NEVER use `---` dividers" constraint

**Risk Level:** Low - Only documentation changes, no code execution

### Phase 2: Test with Manual Generation
1. Manually invoke workflow to generate one test blog post
2. Verify TypeScript syntax is correct (backticks, no parsing errors)
3. Check visual content meets new requirements

**Risk Level:** Medium - Testing GitHub Action, can be rolled back

### Phase 3: Deploy & Monitor
1. Merge changes to main
2. Monitor next automated blog post
3. Validate Vercel build succeeds
4. If issues arise, revert workflow changes

**Risk Level:** Medium - Affects production blog generation

## Risk Mitigation

### TypeScript Syntax Validation
**Risk:** New visual elements could introduce template literal syntax errors
**Mitigation:**
- Workflow prompt already includes TypeScript format section
- Add explicit examples of table formatting in template literals
- Test with typecheck before committing

### GitHub Action Failures
**Risk:** Workflow changes could break the z.ai API integration
**Mitigation:**
- No changes to `.github/workflows/blog-creation-zai.yml` logic
- Only updating prompt text, not workflow structure
- Prompt changes are text-based (low risk)

### Vercel Deploy Failures
**Risk:** Generated blog posts could have syntax errors
**Mitigation:**
- TypeScript template literal syntax already documented
- Tables use standard Markdown table syntax (valid in template literals)
- Test with manual generation before relying on automation

## Success Criteria

1. ✅ No `---` dividers in generated blog posts
2. ✅ At least 2-3 visual elements per post (tables, code blocks, structured sections)
3. ✅ TypeScript syntax is valid (backticks, proper escaping)
4. ✅ Vercel builds succeed with generated content
5. ✅ GitHub Actions complete successfully
6. ✅ Generated posts match visual quality of TikTok guide

## Rollback Plan

If issues occur:
1. Revert `.claude/prompts/blog-creation-workflow.md` to previous version
2. Generated blog posts remain valid (already have proper syntax)
3. Manual fixes to any problematic generated posts

**Recovery Time:** < 5 minutes (git revert workflow file)
