# WARP Scoring Rubric

Detailed 0-25 point scoring criteria for each WARP force. Use this rubric to ensure consistent, evidence-based assessments.

## P: Pervasive Pain (0-25 points)

### Widespread (0-10 points)

| Score | Criteria |
|-------|----------|
| 9-10 | $10B+ TAM, affects 100M+ people globally |
| 7-8 | $1B+ TAM, affects 10M+ people |
| 5-6 | $100M+ TAM, affects 1M+ people |
| 3-4 | $10M+ TAM, affects 100K+ people |
| 1-2 | Niche market, affects <100K people |
| 0 | No clear market size, hobby/interest problem |

### Frequent (0-10 points)

| Score | Criteria |
|-------|----------|
| 9-10 | Multiple times per day, constant friction |
| 7-8 | Daily occurrence |
| 5-6 | Weekly occurrence |
| 3-4 | Monthly occurrence |
| 1-2 | Quarterly or less |
| 0 | Annual or rare occurrence |

### Loud (0-5 points)

| Score | Criteria |
|-------|----------|
| 5 | 100+ active complaints online, trending Reddit threads, viral tweets |
| 4 | 50-100 complaints, strong community discussion |
| 3 | 25-50 complaints, moderate online presence |
| 2 | 10-25 complaints, some discussion |
| 1 | Few isolated complaints |
| 0 | Silent suffering, no visible complaints |

**Evidence sources:**
- Google search: "[problem] sucks", "[competitor] alternative"
- Reddit: Relevant subreddits, upvote counts
- Twitter/X: Complaint search, sentiment
- G2/Trustpilot: Negative reviews mentioning the pain
- Search volume: Google Trends, keyword research

## W: Win Preference (0-25 points)

### Transformation Clarity (0-10 points)

| Score | Criteria |
|-------|----------|
| 9-10 | Crystal clear From→To, users immediately understand the shift |
| 7-8 | Clear transformation, some explanation needed |
| 5-6 | Understandable benefit, requires demo |
| 3-4 | Vague improvement, hard to articulate |
| 1-2 | No clear transformation, incremental improvement |
| 0 | No differentiation from alternatives |

**Test:** Ask 5 potential users to explain what your product does. If they can't articulate the transformation in one sentence, score lower.

### Approach Strength (0-10 points)

| Score | Criteria |
|-------|----------|
| 9-10 | Eliminates 80%+ of old workflow steps, removes expertise entirely |
| 7-8 | Eliminates 50-80% of steps, significantly reduces expertise needed |
| 5-6 | Eliminates 30-50% of steps, some expertise still needed |
| 3-4 | Eliminates 10-30% of steps, mostly optimization |
| 1-2 | Minor efficiency gains, same workflow |
| 0 | No workflow change, just different tool |

**Measure:**
- Time to complete task (old vs new)
- Expertise required (specialist vs anyone)
- Steps in workflow (count reduction)
- Cost to complete (including learning curve)

### Opinionated Defaults (0-5 points)

| Score | Criteria |
|-------|----------|
| 5 | 80%+ of users never change defaults, zero configuration needed |
| 4 | Most users accept defaults, minimal configuration |
| 3 | Some configuration expected, good defaults exist |
| 2 | Significant setup required, weak defaults |
| 1 | Extensive configuration, no meaningful defaults |
| 0 | Every user must configure everything |

**Questions:**
- What decisions do you make FOR the user?
- What % of users can succeed with zero configuration?
- Is customization hidden by default?

## A: Activate Instantly (0-25 points)

### Time-to-Value Scoring

| Score | Time | Criteria |
|-------|------|----------|
| 25 | <30 sec | Value immediate, no friction |
| 22-24 | 30-60 sec | Near-instant value, minimal friction |
| 18-21 | 1-2 min | Quick value, slight friction |
| 15-17 | 2-5 min | Reasonable value, some friction |
| 10-14 | 5-15 min | Slow value, significant friction |
| 5-9 | 15-30 min | Very slow, high friction |
| 0-4 | 30+ min | Too slow for PLG |

### Blocker Assessment

For each blocker type, assess severity:

**Knowledge Gaps (What must users understand?)**
- None: +5 points
- Minimal (one concept): +3 points
- Moderate (2-3 concepts): +1 point
- Significant (4+ concepts): 0 points

**Skill Gaps (What must users execute?)**
- None (AI does it): +5 points
- Minimal (simple actions): +3 points
- Moderate (requires training): +1 point
- Significant (expert skills): 0 points

**Product Gaps (What must users configure?)**
- None (works immediately): +5 points
- Minimal (1-2 settings): +3 points
- Moderate (3-5 settings): +1 point
- Significant (6+ settings): 0 points

**Note:** These are modifiers to the base TTV score, not separate scores.

## R: Repeatable Leverage (0-25 points)

### Zero Marginal Humans (0-10 points)

| Score | Criteria |
|-------|----------|
| 9-10 | 10,000 users = 1 user in support cost, fully self-serve |
| 7-8 | 10,000 users = 10x users in support cost, minimal human touch |
| 5-6 | 10,000 users = 100x users in support cost, some human help needed |
| 3-4 | Linear scaling, every 100 users needs 1 support person |
| 1-2 | Heavy human involvement, services-like model |
| 0 | Cannot scale without proportional headcount |

**The 10,000 User Test:**
> "If 10,000 users signed up tomorrow, could we handle them with current team + infrastructure?"

### AI + Product Automation (0-10 points)

| Score | Criteria |
|-------|----------|
| 9-10 | AI handles onboarding, support, conversion; product improves itself |
| 7-8 | Strong automation, AI assists most journeys |
| 5-6 | Partial automation, humans handle edge cases |
| 3-4 | Limited automation, humans required for success |
| 1-2 | Minimal automation, mostly human-driven |
| 0 | No automation, fully human-driven |

**Assess:**
- Onboarding: Self-serve or sales-led?
- Support: AI chatbot or human team?
- Conversion: Product-led or sales-led?
- Improvement: Product learns or requires engineering?

### Effortless ARR (0-5 points)

| Score | Criteria |
|-------|----------|
| 5 | Self-serve upgrade, viral growth, no sales team |
| 4 | Self-serve with occasional sales assist |
| 3 | Mix of self-serve and sales |
| 2 | Primarily sales-led with some self-serve |
| 1 | Fully sales-led |
| 0 | Enterprise-only, long sales cycles |

**Revenue/Employee Benchmark:**
- $5M+/employee: Exceptional leverage
- $1-5M/employee: Strong leverage
- $500K-1M/employee: Good leverage
- $200-500K/employee: Average
- <$200K/employee: Poor leverage (services-like)

## Overall Score Interpretation

| Total Score | Readiness | Recommendation |
|-------------|-----------|----------------|
| 90-100 | Warp Speed Ready | Go fast, optimize constraint |
| 75-89 | Strong Foundation | Fix constraint, then accelerate |
| 60-74 | Viable PLG | Significant work on 2+ forces |
| 45-59 | Early Stage | Focus on one force at a time |
| <45 | Not Ready | Fundamental model questions |

**Critical Insight:** A 60/100 with one 5/25 force is WORSE than 60/100 with four 15/25 forces. Individual force minimums matter more than the total.

**Minimum thresholds per force:**
- Any force below 10 = Critical weakness
- Any force below 15 = Needs immediate work
- All forces above 15 = Ready for growth focus
