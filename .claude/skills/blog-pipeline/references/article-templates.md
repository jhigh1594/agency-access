# Article Templates

Five article types, each with a distinct structure, word count target, and strategic purpose. Select the type that matches the content calendar assignment. When in doubt, check the calendar — the article type is usually specified.

## 1. Category-Defining

**Purpose:** Own a search term that has no authoritative answer. Define the category and position AuthHub as the expert.

**Word count:** 1,500-2,000 | **Read time:** 8-10 min

**Structure:**
```
Opening: Problem vignette or surprising observation
  → "Every agency does X. Almost none have a system for it."

Section 1: Definition
  → What it is (clear, one-paragraph definition)
  → What it's NOT (distinguish from adjacent concepts like IAM, credential management)
  → Why the category doesn't have a name yet (if applicable)

Section 2: The Cost
  → Quantified impact (hours, dollars, opportunity cost)
  → Real numbers, not vague claims

Section 3: Problems (3-5)
  → Each problem is a short subsection with a concrete scenario
  → Not a numbered list — woven into narrative

Section 4: Framework or Lifecycle
  → Original model the reader can apply
  → Stages, phases, or steps that show the full picture
  → Example: Request → Authorization → Verification → Maintenance → Revocation

Section 5: What Good Looks Like
  → 3-5 characteristics of systems that solve this well
  → Honest about what AuthHub does and doesn't cover

Section 6: When to Use / When to Skip
  → Binary decision framework
  → Not a sales pitch — genuine guidance

Closing: Bookend callback to opening
```

**Reference post:** `what-is-client-access-management.md` (9/10)

## 2. Pillar / Comprehensive Guide

**Purpose:** Deep, authoritative content that serves as the definitive resource on a topic. Long-form, multiple sections, high time-on-page.

**Word count:** 2,500-3,500 | **Read time:** 12-18 min

**Structure:**
```
Opening: Contrarian hook or "real timeline" framing
  → Challenge the conventional wisdom immediately
  → "Every X guide says Y. They skip the part about Z."

Section 1: The Framework (4-6 phases)
  → Original phase-based or step-based model
  → Each phase: what happens, why it matters, how long it takes
  → Day-by-day or week-by-week specificity where possible

Section 2: Common Mistakes (3)
  → What most agencies get wrong
  → Specific scenarios, not abstract advice

Section 3: Data & Evidence
  → 3-5 sourced statistics with attribution
  → Industry benchmarks, survey data, or platform documentation
  → Every stat must have a source or be reframed as observation

Section 4: Timeline Comparison Table
  → Manual process (5-9 days) vs Systematic process (2-4 days)
  → Makes the value tangible and scannable

Section 5: Retention / Success Metrics
  → What to measure after implementing the framework
  → 2-3 specific KPIs with benchmarks

Section 6: Comprehensive Checklist
  → Actionable, checkbox-style
  → Organized by phase from Section 1

Closing: Ties back to the contrarian opening
```

**Reference post:** `how-to-onboard-new-marketing-client.md` (9/10)

## 3. Listicle (Category-Based)

**Purpose:** "Best X for Y" format that stands out by organizing tools into operational categories rather than arbitrary ranking.

**Word count:** 2,000-3,000 | **Read time:** 10-15 min

**Structure:**
```
Opening: Problem-framing with memorable analogy
  → "Like recommending a frying pan, a spatula, and a grocery store
     as alternatives to each other."

Section 1: The Problem Framework (4 operational categories)
  → Define 4-5 distinct problems agencies face
  → Each category has a clear bottleneck it solves
  → This is the organizing principle — not a 1-16 ranking

Section 2: Tools by Category
  → For each category, review 2-4 tools:
    → What it does (one sentence)
    → Pricing (real numbers)
    → Best for (specific agency type/size)
    → "Solves the bottleneck?" (yes/partial/no — honest assessment)
  → Total: 10-14 tools across all categories

Section 3: Evaluation Framework
  → 2-3 questions the reader should ask themselves
  → Not "here's what to buy" — "here's how to decide"

Section 4: Why Lists Get It Wrong
  → Honest meta-commentary about the listicle format itself
  → Positions this article as different from generic competitors

Section 5: Quick-Reference Comparison Table
  → All tools in one scannable table
  → Columns: tool, category, pricing, best for, bottleneck solved

Section 6: Minimum Viable Stack
  → "If you only have budget for X tools, here's what to pick"
  → Honest recommendation based on highest-impact bottleneck
```

**Reference post:** `best-client-onboarding-software-agencies-2026.md` (9/10)

## 4. Tutorial / Platform Guide

**Purpose:** Step-by-step instructions for getting platform access. Both the manual process and the AuthHub alternative.

**Word count:** 1,800-2,500 | **Read time:** 9-12 min

**Structure:**
```
Opening: The pain point
  → "You've been there. The client says they granted access.
     The platform says they didn't."

Section 1: Quick Answer (featured snippet target)
  → 2-3 sentence direct answer
  → Structured for Google's featured snippet extraction

Section 2: Platform Overview
  → Account structure / hierarchy
  → Permission levels table
  → What's different about THIS platform (unique challenges)

Section 3: Manual Process (step-by-step)
  → Numbered steps with specific UI paths
  → Include email template where applicable
  → Screenshots referenced but not embedded (markdown)

Section 4: AuthHub Alternative (step-by-step)
  → Shorter than manual process (the "5 minutes" promise)
  → Clear comparison: "Instead of X, you do Y"

Section 5: Common Problems Table
  → 4-6 issues with symptoms, causes, and fixes
  → Scannable table format

Section 6: Security Best Practices
  → 4-6 practices specific to this platform
  → SOC2 compliance mention where relevant

Section 7: Pro Tips (3-5)
  → Advanced tips for experienced users
  → Not generic advice — platform-specific

Section 8: Scaling
  → How to handle this at volume (10+ clients)
  → AuthHub mention here is natural

Key Takeaways (optional)
```

**Reference post:** `linkedin-ads-access-agency.md` (7/10), `tiktok-ads-access-agency.md` (7/10)

## 5. Comparison / Data-Driven Analysis

**Purpose:** Head-to-head comparison using real data. Competitive positioning with honest assessment.

**Word count:** 1,500-2,500 | **Read time:** 8-12 min

**Structure:**
```
Opening: Vignette that illustrates the decision
  → Real scenario: "The invoice shows $79 for a month you expected to pay $49."

Section 1: Model Definitions
  → Clear explanation of each model being compared
  → No loaded language — let the data speak

Section 2: Scenario Analysis (3 scenarios)
  → Small agency (3-5 clients)
  → Mid agency (10-15 clients)
  → Large agency (25+ clients)
  → Each scenario: table with real cost calculations

Section 3: The Surprise Scenario
  → The case where the "obvious" choice goes wrong
  → Specific numbers: "$150 surprise" style

Section 4: When Each Model Works
  → Honest assessment of where the competitor's model wins
  → Builds credibility — this isn't an advertorial

Section 5: Evaluation Checklist
  → 4-6 questions to help the reader decide for their situation

Closing: Clear recommendation tied to agency size/type
```

**Reference post:** `flat-rate-vs-credit-pricing.md` (9/10)
